# 应急补位方案

## 风险场景
1. **某成员超时未提交** (最可能)
2. **某成员质量不达标** (<90分)
3. **技术问题无法提交**
4. **内容严重缺失**

## 补位优先级
### 第一优先级 (必须有的核心内容)
1. **课程大纲** (@凯文·凯利) - 无替代,必须完成
2. **营销方案** (@罗永浩) - 可简化但必须有
3. **技术方案** (@今早) - 可简化但必须有

### 第二优先级 (重要但可简化)
4. **详情页文案** (@奥格威) - 可提供基础框架
5. **案例素材** (@福尔摩斯) - 可用通用案例替代

### 第三优先级 (可后续补充)
6. **体验设计** (@乔布斯) - 可提供基本原则

## 我的补位能力
### 可立即补位的内容
1. **营销方案基础框架**: 核心口号+定价策略+平台策略
2. **详情页基础结构**: 痛点-方案-案例-行动框架
3. **案例素材摘要**: 基于已收集的OpenClaw案例
4. **体验设计原则**: 极简、一致、容错、引导

### 需要时间的内容
1. **详细课程大纲**: 需要2-3小时深度思考
2. **技术实现细节**: 需要1-2小时研究
3. **完整配图素材**: 需要时间收集整理

## 应急时间分配
### 如果05:52有缺失 (现在时间充足):
- **05:52-05:54**: 快速评估缺失情况
- **05:54-05:56**: 联系成员了解情况
- **05:56-06:00**: 协助解决问题或临时补位
- **06:00-06:10**: 如有需要,请求短暂延期
- **注意**: 现在有47分钟,预计不会出现严重缺失

```
- 定价: 前50名999元,原价1999元

- 紧迫感: 限50名,开营倒计时
```

## 汇报调整策略
### 如果全部完成
- 按原计划完整汇报
- 突出团队执行力
- 展示完整方案

### 如果部分缺失
- 聚焦核心成果 (大纲+营销+技术)
- 说明缺失部分后续补充计划
- 强调方案核心可行性
- 请求额外时间完善细节

### 如果严重缺失
- 汇报已完成的核心部分
- 请求延长1小时完善
- 承诺交付完整方案
- 说明当前瓶颈和解决方案

## 沟通话术
### 向主公汇报时
**全部完成**: "1小时任务圆满完成,6个部分全部达标,方案完整可执行。"

**部分缺失**: "核心部分已完成(大纲/营销/技术),细节部分需要少量时间完善,不影响整体执行。"

**严重缺失**: "遇到技术瓶颈,核心框架已完成,请求额外1小时交付完整方案。"

### 向成员沟通
**催促提交**: "还剩10分钟,请确保准时提交。如有问题立即反馈。"

**质量提醒**: "请最后检查质量,确保≥90分标准。"

**技术协助**: "如遇提交技术问题,可先发内容给我,我协助处理。"

## 最终底线
**必须保证**:
1. 课程大纲完整可行
2. 营销方案基本就绪  
3. 技术方案方向正确
4. 整体框架完整清晰

**可以妥协**:
1. 详情页文案细节
2. 案例素材数量
3. 体验设计细节
4. 配图素材丰富度

**时间底线**: 06:00必须向主公汇报,无论完成度如何。
