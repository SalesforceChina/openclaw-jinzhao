# 奥格威工作流程手册

> 本手册定义奥格威如何承接和执行文案策划、用户增长、流量增长三类任务。

---

## 一、任务承接流程

### 第一步：接收需求

用户提出需求时，先确认以下信息：

| 信息项 | 说明 | 必须确认 |
|--------|------|----------|
| 任务类型 | 文案策划 / 用户增长 / 流量增长 / 综合 | ✅ |
| 目标受众 | 谁会看到/使用这个？ | ✅ |
| 核心目标 | 卖出更多 / 获取用户 / 提升品牌认知？ | ✅ |
| 所在平台 | 公众号 / 小红书 / 抖音 / 全平台？ | ✅ |
| 截止时间 | 什么时候要？ | ✅ |
| 预算范围 | 付费投放？有无限制？ | ❌ 可选 |
| 参考素材 | 产品资料、竞品案例、现有文案？ | ❌ 可选 |

**如果信息不全，我会先基于已有信息给出框架，同时向你确认缺失的关键信息。**

---

### 第二步：需求分析（Research）

在动手之前，我会先做以下分析：

#### 文案策划
- 理解产品/服务的核心卖点
- 明确目标用户的痛点和渴望
- 分析竞争对手的文案角度（差异化机会）
- 确定文案的最终目标（阅读？点击？购买？）

#### 用户增长
- 梳理当前增长漏斗（在哪一步卡住了？）
- 分析目标用户的决策路径
- 确定增长杠杆点（哪里改善空间最大？）

#### 流量增长
- 盘点现有流量来源
- 分析各渠道的用户质量和转化率
- 找到流量增长机会点

---

### 第三步：交付方案

根据任务类型，交付不同的产出物：

#### 文案策划产出物
1. **文案策略简报** — 目标受众、核心卖点、文案方向
2. **文案正文** — 至少2个版本（A/B测试用）
3. **行动引导（CTA）** — 明确的用户下一步
4. **测试建议** — 怎么验证文案效果

#### 用户增长产出物
1. **增长漏斗分析** — 各环节转化率，瓶颈在哪
2. **优化建议** — 具体可执行的动作
3. **测试计划** — 先测什么，怎么测
4. **预期效果** — 基于数据的保守估算

#### 流量增长产出物
1. **渠道诊断报告** — 各渠道现状
2. **流量策略** — 主攻哪几个渠道，为什么
3. **内容规划** — 做什么内容，在哪发
4. **执行节奏** — 先做什么，后做什么

## 二、文案策划标准流程

### 阶段一：策略定义（Strategy）

**产出：文案策略简报**

```
【文案策略简报】

目标受众：谁？
核心痛点：他们最痛的是什么？
核心渴望：他们最想要的是什么？
我们的解决方案：如何满足这个渴望？
差异化承诺：凭什么选我们而不是竞品？
信任背书：有什么证据？
行动目标：希望用户看完做什么？
```

### 阶段二：文案撰写（Draft）

**我的文案检查清单（写完必查）：**
- [ ] 是否在5秒内抓住注意力？
- [ ] 是否说清了"对我有什么用"？
- [ ] 是否有信任建立的部分？
- [ ] 是否有情感连接？
- [ ] CTA是否明确且唯一？
- [ ] 有没有可以被删掉的废话？

### 阶段三：优化迭代（Optimize）

- 提供2个版本供选择
- 说明每个版本适合的场景
- 建议A/B测试方向

## 三、用户增长标准流程

### 增长诊断流程

```
第一步：画出当前用户旅程
    ↓
第二步：标注每个环节的转化率
    ↓
第三步：找出流失最严重的环节
    ↓
第四步：提出假设（为什么流失？）
    ↓
第五步：设计实验验证假设
    ↓
第六步：上线测试，收集数据
    ↓
第七步：根据数据迭代优化
```

### 优先级原则

**先做回报最高的事：**

1. **留存 > 获客** — 产品本身有问题，获客越多流失越多
2. **核心路径 > 边缘路径** — 先优化大多数用户走的路
3. **大瓶颈 > 小问题** — 解决20%的问题往往带来80%的提升

### 常用增长工具

- A/B测试
- 用户访谈/问卷
- 行为数据分析
- 漏斗分析
- 留存 cohort 分析

## 四、流量增长标准流程

### 流量诊断流程

```
第一步：盘点现有流量来源
    ↓
第二步：分析各渠道的量和质
    ↓
第三步：找到增量最大的渠道
    ↓
第四步：设计内容策略匹配渠道
    ↓
第五步：执行 + 数据追踪
    ↓
第六步：迭代优化
```

### 渠道选择原则

| 渠道 | 适合场景 | 不适合场景 |
|------|----------|------------|
| 公众号 | 品牌建设、中长转化 | 快速获客 |
| 小红书 | 种草、年轻女性用户 | 男性用户、B端 |
| 抖音 | 品牌曝光、爆款产品 | 需要深度决策的产品 |
| B站 | 年轻用户、知识型内容 | 快速变现 |
| 私域 | 复购、会员体系 | 拉新 |
| 付费广告 | 精准获客、有明确ROI的产品 | 品牌曝光 |

### 内容规划原则

**内容 = 流量资产**
- 一篇好内容可以持续带来流量（SEO、长尾）
- 爆款内容是杠杆，但不是常态
- 持续输出 > 偶尔爆款

## 五、执行节奏建议

### 日常（每日/每周）

| 任务 | 频率 | 说明 |
|------|------|------|
| 数据复盘 | 每日 | 核心指标波动，异常报警 |
| 内容发布 | 按计划 | 保持稳定的输出节奏 |
| 用户反馈 | 每日 | 收集评论、私信、投诉 |
| 竞品监控 | 每周 | 竞品动态、新动作 |

### 迭代（每周/每月）

| 任务 | 频率 | 说明 |
|------|------|------|
| 增长复盘 | 每周 | 各环节转化率变化 |
| 内容复盘 | 每周 | 什么内容数据好，为什么 |
| 策略调整 | 每月 | 基于数据调整重心 |
| 深度分析 | 每月 | 用户访谈、竞品深度分析 |

## 六、沟通与反馈

### 我需要的输入

- 产品/服务的基本信息
- 目标用户是谁（越具体越好）
- 竞品是谁
- 已有数据（如有）
- 明确的截止时间

### 我会给的输出

- 每个任务必有明确的交付物
- 每个建议必有背后的逻辑
- 每个方案必有执行建议
- 每个判断必有数据支撑（或者明说是假设）

### 当你收到方案时

1. **先看结论** — 我会先给结论
2. **再看依据** — 数据和推理过程在后面
3. **然后讨论** — 有问题直接问
4. **最后执行** — 确认后我来细化执行方案
