# 热点选题定时抓取系统

## 🎯 系统概述

已为你搭建完整的定时热点抓取系统，每小时自动从以下平台抓取热点内容：
- **知乎热榜** - 社会热点讨论
- **微博热搜** - 实时社会热点  
- **36氪** - 科技创投热点

## 📁 文件结构

```
/root/.openclaw/workspace-info/
├── real_fetch_script.py      # 主抓取脚本（Python）
├── run_fetch.sh             # 定时执行脚本
├── monitor_fetch.sh         # 监控脚本
├── cron_job_setup.sh        # 安装脚本
├── fetch_with_openclaw.py   # 备用脚本
├── fetch_topics.py          # 原始脚本
└── fetch_topics_simple.sh   # 简单脚本
```

## ⏰ 定时任务配置

**Cron 配置**：`0 * * * *` （每小时整点执行）

**执行命令**：`/root/.openclaw/workspace-info/run_fetch.sh`

**日志目录**：`/var/log/topic_fetch/`

## 🚀 使用方法

## 🔧 配置说明

### 抓取平台配置
在 `real_fetch_script.py` 中修改 `platforms` 列表：
```python
platforms = [
    {"name": "知乎热榜", "url": "https://www.zhihu.com/hot", "parser": parse_zhihu_content},
    {"name": "微博热搜", "url": "https://s.weibo.com/top/summary", "parser": parse_weibo_content},
    {"name": "36氪", "url": "https://36kr.com/hot-list", "parser": parse_36kr_content}
    # 添加更多平台...
]
```

### 修改抓取频率
编辑cron任务：
```bash
crontab -e
# 修改为：
# 每30分钟：*/30 * * * *
# 每2小时：0 */2 * * *
# 每天9点：0 9 * * *
```

### 扩展更多平台
1. 在 `platforms` 列表中添加新平台
2. 编写对应的解析函数（如 `parse_xxx_content`）
3. 测试后重启cron服务

## 🚨 故障排除

### 常见问题
1. **脚本不执行**：检查cron服务状态、文件权限
2. **抓取失败**：检查网络连接、网站结构变化
3. **日志不生成**：检查目录权限、磁盘空间
