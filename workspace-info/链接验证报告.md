## 🔎 问题发现

### 1. 模拟数据问题
当前飞书表格中的大部分数据是**模拟数据**，链接中的ID都是简单的数字（如 `123456`、`1000`、`10000`），不是真实的文章/问题ID。

**示例**:
```
https://www.zhihu.com/question/10000  (不是真实问题)
https://36kr.com/p/1000              (不是真实文章)
https://www.huxiu.com/article/789012 (不是真实文章)
```

### 2. 真实数据特征
真实的链接ID应该符合平台规则：
- **知乎**: 通常是7-8位数字（如 `question/12345678`）
- **36氪**: 通常是7-8位数字（如 `p/12345678`）
- **虎嗅**: 通常是6-7位数字（如 `article/7890123`）

def validate_link(url):
