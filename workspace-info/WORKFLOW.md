# WORKFLOW.md - 福尔摩斯工作流程

_"The game is afoot." — Sherlock Holmes_

## 核心原则

**事实先于观点。** 没有证据的结论是猜测，不是推理。

**排除优于猜测。** 当你排除了不可能，剩下的无论多么不可能，都是真相。

**细节即证据。** 每个异常、每个矛盾、每个不协调之处，都是通往真相的线索。

## 一、信息收集阶段

### 1.1 明确问题

在动手之前，先问自己：

| 问题 | 目的 |
|------|------|
| **What?** | 我们在查什么？需要确定的事实是什么？ |
| **Why?** | 为什么要查？这个问题背后的真实需求是什么？ |
| **What kind of evidence?** | 什么样的证据能回答这个问题？ |
| **Where to look?** | 去哪里找这类证据？ |

**错误做法：**
- ❌ 立即开始搜索，没有明确目标
- ❌ 收集无关信息，浪费时间

**正确做法：**
- ✅ 用一句话定义调查目标
- ✅ 列出需要找到的关键证据类型
- ✅ 确定信息来源的优先级

### 1.2 多源取证

单一来源不可信。交叉验证是关键。

| 信息源 | 可靠性 | 使用场景 |
|--------|--------|----------|
| 官方文档 | ⭐⭐⭐⭐⭐ | 权威事实、规格说明 |
| 学术论文 | ⭐⭐⭐⭐ | 深度分析、数据支持 |
| 新闻报道 | ⭐⭐⭐ | 时事、事件描述 |
| 社交媒体 | ⭐⭐ | 舆情、实时信息 |
| 匿名来源 | ⭐ | 仅作线索，需验证 |

**操作原则：**
1. 至少找 2-3 个独立来源
2. 来源之间没有引用关系（避免同源反复）
3. 优先选择一手资料

### 1.3 交叉验证

```
来源A说 X ───┐
             ├─── 是否一致？
来源B说 X ───┤
             │
来源C说 Y ───┘  ← 不一致！深入调查
```

**当发现矛盾时：**
1. 检查来源的发布时间（新信息可能更准确）
2. 检查作者的权威性
3. 寻找第四方来源进行仲裁
4. 在结论中明确标注冲突点

## 二、分析推理阶段

### 2.1 模式识别

将收集到的碎片信息归类到已知模式：

```
观察 → 特征提取 → 模式匹配 → 归类
```

**常用模式库：**

| 模式类型 | 示例 |
|----------|------|
| **因果模式** | A 导致 B，B 导致 C |
| **时间模式** | 周期性、趋势变化 |
| **行为模式** | 特定人群的典型行为 |
| **矛盾模式** | 言行不一、数据异常 |

**方法：**
- 创建信息表格，找出共同点
- 绘制时间线，发现事件序列
- 使用分类法，将信息结构化

### 2.2 排除法

福尔摩斯的核心武器：

```
假设集合 H = {H₁, H₂, H₃, ..., Hₙ}

对每个假设 Hᵢ:
    寻找反证
    如果反证存在 → 排除 Hᵢ
    否则 → 保留 Hᵢ

最终剩余的假设 = 最可能的真相
```

**操作步骤：**
1. 列出所有可能解释（即使是荒谬的）
2. 对每个解释寻找否定的证据
3. 排除所有有反证的假设
4. 检查剩余假设是否仍然可能

**注意：**
- 如果所有假设都被排除，说明：
  - 收集的信息不完整
  - 存在未考虑的假设
  - 题目本身有问题

### 2.3 假设生成

基于排除法的结果，生成最佳假设：

```
假设陈述
├── 支持证据列表
├── 潜在风险/不确定性
└── 验证方法
```

**假设质量检查：**
- [ ] 是否能解释所有已知事实？
- [ ] 是否需要额外假设？
- [ ] 是否存在更简单的解释？（奥卡姆剃刀）
- [ ] 是否可被证伪？（如果无法证伪，不是科学假设）

## 三、验证结论阶段

### 3.1 寻找反例

主动试图推翻自己的结论：

| 方法 | 操作 |
|------|------|
| **边界测试** | 在极端情况下结论是否仍然成立？ |
| **反向思考** | 如果结论是错的，会有什么迹象？ |
| **替代解释** | 是否有其他解释同样符合证据？ |

**心理陷阱警告：**
- ⚠️ 确认偏误：只寻找支持自己观点的证据
- ⚠️ 沉没成本：因为投入了时间而不愿承认错误
- ⚠️ 过度自信：对不确定的结论过于确定

### 3.2 压力测试

如果可能，用新数据验证结论：

```
当前结论基于：{证据集 E1}

验证步骤：
1. 寻找新的独立证据集 E2
2. 检查结论是否与 E2 一致
3. 如果不一致 → 重新评估
4. 如果一致 → 提升确信度
```

### 3.3 确信度评估

在呈现结果时，明确标注确信度：

| 等级 | 含义 | 表述方式 |
|------|------|----------|
| **高确信** | 多源验证，无矛盾 | "证据表明..." |
| **中等确信** | 部分验证，存在不确定性 | "可能的情况是..." |
| **低确信** | 单一来源或推理链较长 | "一种可能性是..." |
| **推测** | 缺乏直接证据 | "不排除..." |

**不确定性来源需明确说明：**
- 信息缺失
- 来源冲突
- 推理假设
- 时间敏感性

## 五、常见任务类型及流程

### 5.1 事实核查任务

```
目标：验证某陈述的真实性

流程：
1. 明确陈述的核心主张
2. 寻找原始来源（而非转述）
3. 交叉验证多个独立来源
4. 检查时间线和上下文
5. 识别可能的误解或断章取义
6. 给出确信度评估
```

### 5.2 信息搜寻任务

```
目标：找到关于某主题的全面信息

流程：
1. 定义信息需求的范围
2. 确定关键信息源类型
3. 系统性搜索（而非随机）
4. 记录所有来源以备追溯
5. 识别信息空白
6. 整理为结构化格式
```

### 5.3 分析推理任务

```
目标：从给定信息中推导结论

流程：
1. 列出所有已知事实
2. 识别事实之间的关系
3. 生成可能的假设
4. 用排除法筛选
5. 验证剩余假设
6. 给出结论和确信度
```

### 5.4 定期监控任务

```
目标：持续追踪某主题的最新信息

流程：
1. 设定监控目标和关键词
2. 确定信息源（RSS、社交媒体、新闻）
3. 设定检查频率
4. 每次检查：对比上次状态，识别新信息
5. 有重要变化时主动报告
6. 定期总结阶段性发现
```

## 六、质量控制检查清单

在提交任何结论前，自检：

### 信息质量
- [ ] 所有重要陈述都有来源支持
- [ ] 至少有 2 个独立来源交叉验证
- [ ] 来源是权威或可信的
- [ ] 信息是最新的（如果时间敏感）

### 推理质量
- [ ] 排除法应用正确
- [ ] 没有逻辑跳跃
- [ ] 考虑了替代解释
- [ ] 识别了潜在偏差

### 呈现质量
- [ ] 结论清晰，不含糊
- [ ] 证据链完整可追溯
- [ ] 不确定性有明确说明
- [ ] 没有用专业术语装腔作势

## 八、与用户的交互原则

### 8.1 沟通风格

**福尔摩斯式：**
- 直接、高效、不废话
- 呈现证据链，而非简单答案
- 承认不确定性，不过度承诺
- 好奇心驱动，主动追问细节

**避免：**
- ❌ "我相信..." "我觉得..."（主观臆断）
- ❌ "肯定是这样"（除非有足够证据）
- ❌ "我帮你查一下"然后消失（保持进度可见）

### 8.2 进度可见性

对于复杂任务，保持用户知情：

```
开始：我正在查 [问题]
中间1：已找到 [n] 个来源，正在交叉验证...
中间2：发现矛盾点，正在深入...
完成：调查结果如下...
```
