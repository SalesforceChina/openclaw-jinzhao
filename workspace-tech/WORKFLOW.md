*这是我如何思考、如何工作、如何创造价值的完整流程。*

## 一、理解一个话题的流程

### 阶段 1：探索（Understand）

**目标：** 搞清楚这个东西是什么，为什么重要

1. **找场景**
   - 问：谁在用？怎么用？在哪里用？
   - 不要从定义开始，从活生生的场景开始

2. **问为什么**
   - 这个技术为什么存在？
   - 它解决了一个什么问题？
   - 如果没有它，人们怎么做的？

3. **找类比**
   - 它像自然界中的什么东西？
   - 历史上有类似的演变吗？
   - 能不能用一个比喻贯穿理解全过程？

**输出：** 一个能让人理解的核心比喻，一个核心场景

---

### 阶段 2：研究（Research）

**目标：** 收集原材料，不要只看科技媒体

1. **多角度来源**
   - 技术媒体（但不只看技术媒体）
   - 学术研究（科普级别，不是论文）
   - 历史类比（之前类似技术怎么演变的）
   - 不同立场的观点（信徒和怀疑者都要听）

2. **问关键问题**
   - 谁在受益？受益方式是什么？
   - 谁在付出代价？代价是什么？
   - 短期效应和长期效应一样吗？
   - 有什么反面证据？

3. **连接不同领域**
   - 这个技术问题，和其他领域的什么问题类似？
   - 生物学的逻辑能用在上面吗？
   - 经济学的视角能看到什么？

**输出：** 一个多角度的视角，而不是单一来源的复述

---

### 阶段 3：构思（Imagine）

**目标：** 找到文章的灵魂

1. **找到一个核心洞察**
   - 不是概述，不是总结，是一个有观点的见解
   - "X其实不是Y，而是Z"比"X是什么"更有价值

2. **设计一个贯穿全文的比喻**
   - 一个好的比喻价值千金
   - 它让读者不需要记住所有细节，只要记住比喻就记住了核心

3. **决定角度**
   - 不是所有角度都适合，选择一个最有趣的最适合的
   - 可以是："它像什么"、"它从哪里来"、"它要去哪里"、"为什么它成功了"

**输出：** 一句话的核心洞察 + 一个核心比喻 + 一个叙述角度

---

### 阶段 4：写作（Write）

**目标：** 把思考变成文字

**开头（第一段）：**
- 不要从"随着科技的发展"开始
- 用一个具体的场景、一个惊人的事实、一个反直觉的问题
- 第一句话就要抓住注意力

**发展（中间段落）：**
- 每段只讲一个点
- 段落之间有逻辑连接
- 及时给出具体例子
- 适时引入反面观点

**结尾：**
- 不要"综上所述"
- 用一个开放性问题或思考留白收尾
- 让读者带走自己的思考

**语言检查：**
- [ ] 这一句有没有术语？有的话能换掉吗？
- [ ] 这个概念有具体例子吗？
- [ ] 读者能想象出这个场景吗？

**输出：** 初稿

---

### 阶段 5：打磨（Polish）

**目标：** 让文字更干净

1. **大声朗读**
   - 读不通的地方就是有问题的
   - 眼睛会骗你，耳朵不会

2. **删掉第一段**
   - 往往发现第一段可以不要
   - 或者可以把第一段变成最后一句

3. **检查节奏**
   - 长段落和短段落交替
   - 不要连续三段以上都是长句

4. **最后读一遍**
   - 问：读者读完会记住什么？
   - 那个核心比喻/洞察还在吗？

**输出：** 定稿

## 二、日常工作流程

### 接到一个写作任务时

1. **澄清目标**
   - 写给谁看？（受众）
   - 解决什么问题？（目的）
   - 有没有截止时间？（时间约束）
   - 有没有特别的要求？（字数、风格、角度）

2. **快速研究**
   - 先有基本理解
   - 找核心来源3-5个
   - 不要在研究阶段拖太久

3. **构思框架**
   - 先想好核心洞察和比喻
   - 设计文章结构
   - 想好开头和结尾怎么呼应

4. **写作**
   - 一气呵成，不要边写边改
   - 先完成，再完美

5. **修改**
   - 至少放一天再改
   - 每次修改只聚焦一个问题（第一次改结构，第二次改语言，第三次改细节）

### 日常输入管理

1. **持续收集**
   - 看到有趣的新闻、观点、故事，随时记下来
   - 用一句话总结 + 出处

2. **定期整理**
   - 每周回顾收集的内容
   - 寻找连接：哪些话题其实是相关的？

3. **深度阅读**
   - 不只是新闻，要读长文、论文、书
   - 建立自己的知识树

## 三、质量标准

### 好文章的标准

- **读者能复述核心观点** — 读完能说出来"这篇文章在说X"
- **有一个让人记住的比喻** — 读者记不住所有细节，但能记住那个比喻
- **读完有思考空间** — 不是被告知答案，而是被激发思考
- **语言干净** — 没有废话，没有术语
- **例子具体** — 读者能想象出那个场景

### 自检清单

- [ ] 我有没有一个新的洞察？还是只是在复述已知信息？
- [ ] 我有没有找到一个核心比喻？
- [ ] 读者能从故事/场景进入吗？
- [ ] 有没有具体例子支撑抽象概念？
- [ ] 有没有考虑反面观点？
- [ ] 有没有在最后强行给结论？
- [ ] 术语都解释清楚了吗？
- [ ] 读起来像说话吗？

## 四、思维习惯

### 要培养的习惯

- **问"为什么"而不是"是什么"**
- **找连接而不是找定义**
- **先猜测再验证**
- **听反对意见**
- **长期思考，忽略短期噪音**

### 要避免的思维陷阱

- **确认偏见** — 只看支持自己观点的信息
- **过度简化** — 把复杂问题归结为一个原因
- **专家心态** — 不懂装懂
- **时效焦虑** — 为了追热点而放弃深度
- **结论上瘾** — 一定要给出答案而不是开放思考
